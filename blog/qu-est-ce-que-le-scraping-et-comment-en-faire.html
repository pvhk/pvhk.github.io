Qu'est ce que le scrapping et comment en faire ?

Le scraping de données ou “web scraping” est une technique permettant de récupérer des données sur un site internet, et de les structurer. Le but est, après extraction et structuration des données, de pouvoir les exploiter et réutiliser ces données en fonction de notre usage (par exemple, constituer une base de données ou faire de la veille concurrentielle). Il est possible de faire du scraping “one shot”, ce qui veut dire que l’on récolte une seule fois des données, pour par exemple créer une base de données. Mais il est également possible de faire du scraping récurrent, par exemple récolter quotidiennement afin d’avoir des données à jour.

Cas d’usage du scraping

Veille concurrentielle

Le web scraping permet de garder un oeil sur vos concurrents. Vous pouvez récolter une multitude de données, par exemple, pour un site e-commerce, il peut être utile de scraper quotidiennement le prix de certains produits chez les sites concurrents, afin d’adapter les prix et être le plus compétitif possible.

Il est également possible de personnaliser ce processus pour automatiquement détecter les variations de prix chez un concurrent, et avoir une alerte quand celui-ci change un prix.

Récolte de leads

Le scraping de données peut être très utile dans la récolte de leads pour remplir un CRM afin d’avoir une liste structurée avec les informations nécessaires pour contacter vos potentiels clients.

Par exemple, récupérer l’ensemble des médecins ou hôpitaux de la ville de Paris. En parcourant plusieurs sites web, il est possible pour des scraper de récupérer les nom, prénoms, adresses mails, numéro de téléphone etc… et de les structurer pour les restituer sous forme de fichiers structurés (CSV, JSON, XLSX etc…) ou directement les ajouter via l’API d’un CRM.

Construction d’une base de données

Vous pouvez également utiliser le web scraping pour construire une base de données afin d’utiliser ces données pour votre propre site web, ou alors pour entrainer des algorithmes de machine learning.

Par exemple, vous souhaitez afficher tous les hotels de Paris sur votre site internet. Il faudra pour cela récolter l’ensemble des hotels de Paris (par exemple, sur google maps ou trip advisor) et les ajouter dans une base de données exploitable par votre site web (mySQL par exemple). Il est ensuite possible d’utiliser un algorithme permettant de régulièrement ajouter ou supprimer les nouveaux hotels dans votre base de données.

Analyse de l’e-reputation

Il est aussi possible d’utiliser le scraping de données pour ensuite passer un algorithme d’analyse sur ces données. Par exemple, monitorer votre e-reputation en récoltant les articles ou tweets au sujet de votre marque ou votre produit, puis passer des algorithmes de NLP afin d’en sortir des thématiques ou des polarités afin de garder un oeil sur son image de marque.

Comment faire du web scraping ?

Les outils de scraping existants

Il est possible de faire du web scraping en utilisant des outils déjà existants. Il en existe une multitude qui fonctionnent bien (par exemple, l’extension chrome “web scraper” : https://chrome.google.com/webstore/detail/web-scraper-free-web-scra ou alors, le service “zyte” : https://www.zyte.com/)

Les soucis de ces outils est qu’ils peuvent être difficiles à prendre en main et/ou ne pourront pas forcément fonctionner sur des sites un petit peu complexes. Mais il existe beaucoup de cas d’usages où ils répondront parfaitement au besoin !

Développer vos propres scrapers

Si vous avez des développeurs en interne, il est possible pour eux de développer des scrapers pour votre besoin. L’avantage d’avoir des scrapers sur-mesures, est qu’ils répondent parfaitement à votre besoin et peuvent potentiellement être modifiés par la suite si jamais il faut des évolutions. Il existe des technologies déjà existantes permettant de faciliter le développement de ceux-ci, par exemple, Scrapy.

Le problème de cette technique est que cela va bloquer vos développeurs pour un moment, et cela peut également être assez compliqué, le web scraping ayant pas mal de subtilités (sécurité des sites web, gestion du javascript etc…)

Faire appel à une agence spécialisée

La dernière alternative est de faire appel à une agence dont c’est le métier (nous par exemple !). Le scraping de page web est il légal ?

La question de la légalité revient souvent quand on parle de ce sujet, car cela peut s’apparenter à du vol, pour faire court : Oui il est légal de scraper de la donnée publique sur un site web, tant qu’il ne faut pas se connecter avec un compte (et accepter des CGU) pour accéder à ces données.

Si jamais il faut se connecter pour accéder à ces données, cela se fait au cas par cas, en fonction des CGU du site et de la nature des données.

D’ailleurs, pour l’anecdote, Google utilise le scraping pour ranker les sites sur le moteur de recherche ! Donc pas d’inquiétudes à avoir sur ce sujet.
